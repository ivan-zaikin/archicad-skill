# Archicad 25 JSON API

Скилл для чтения данных и расчётов по открытому проекту Archicad 25.
API работает только когда Archicad запущен: `POST http://127.0.0.1:19723`
с телом `{"command": "API.<Имя>", "parameters": {...}}`.

## С чего начать

1. Проверь соединение: `python scripts/ac.py info` — вернёт версию и сборку.
   Если ошибка — попроси пользователя запустить Archicad и открыть проект.
2. Для обзора проекта: `python scripts/ac.py types` — количество элементов по типам.
3. **Перед любой ведомостью с зонами:** `python scripts/ac.py validate-zones` —
   расхождение NetArea/CalculatedArea > 1% означает устаревшие зоны; скажи
   пользователю пересчитать (Документ → Обновить → Зоны) до начала расчётов.
4. **Этажи:** используй `python scripts/ac.py stories` вместо вычисления
   разности отметок вручную — команда возвращает готовый список с elevation.
5. Дальше выбирай инструмент под задачу (см. ниже).

## Инструменты

`scripts/ac.py` — клиент без внешних зависимостей, вывод всегда JSON:

```
python scripts/ac.py info                        # версия Archicad
python scripts/ac.py types                       # элементы по типам
python scripts/ac.py call API.GetAllElements     # произвольная команда
python scripts/ac.py call API.GetElementsByType "{\"elementType\": \"Zone\"}"
python scripts/ac.py find-prop Volume            # поиск встроенных свойств
python scripts/ac.py values Wall General_NetVolume,General_Area > walls.json
python scripts/ac.py values-for GUID1,GUID2 General_Area,Construction_CompositeName
python scripts/ac.py stories                     # список этажей из отметок стен
python scripts/ac.py validate-zones              # кросс-проверка зон перед ведомостью
python scripts/ac.py zone-geometry               # bbox-геометрия зон: fill_ratio, height_mismatch
python scripts/ac.py zone-categories             # имена + коды категорий зон (расшифровка ZoneCategoryCode)
```

`values` — главная рабочая лошадка для сводок: выгружает значения встроенных
свойств для всех элементов типа. Дальше агрегируй полученный JSON
коротким Python-скриптом (суммы, группировки, таблицы).

`values-for` — то же, но для **конкретных GUID** (одна зона, найденные колонны,
отфильтрованные стены): первый аргумент — список guid через запятую, `@файл`
или `-` (stdin). Не пиши одноразовый Python ради чтения свойств у горстки
элементов — `values-for` уже делает правильно две вещи, на которых легко
споткнуться вручную: берёт ответ из ключа `propertyValuesForElements`
(не `propertyValues`) и разворачивает перечисления. Нераспознанные имена
свойств обе команды пропускают с предупреждением в stderr, а не валят весь
запрос (имена сверяй через `find-prop`).

Для сложных запросов (фильтрация по этажам, связки зон с элементами,
классификации) пиши Python-скрипт, импортируя функции из ac.py:
`from ac import call, get_property_ids`. В долгих пакетных скриптах вместо
`call` бери `call_with_retry(command, params, attempts=5, delay=4)` — при
открытом в Archicad диалоге (ошибка 4001) она повторяет запрос, давая время
закрыть диалог, вместо немедленного падения.

**Если Python недоступен** — `scripts/ac.ps1`: PowerShell-порт с теми же
командами и форматом вывода (info, call, types, find-prop, values), работает
на голой Windows без установки чего-либо:

```
pwsh -File scripts/ac.ps1 values Wall General_NetVolume,General_Area
```

Все рецепты переносятся 1:1 (замени `python scripts/ac.py` на
`./ac.ps1`). Агрегацию делай средствами PowerShell: `ConvertFrom-Json`,
`Group-Object`, `Measure-Object -Sum`. Для разовых сырых вызовов хватает
и curl: `curl -X POST http://127.0.0.1:19723 -H "Content-Type: application/json" -d "{\"command\":\"API.GetProductInfo\"}"`.

## Справочники (читай по необходимости)

- `references/commands.md` — все 46 команд AC25 с параметрами и JSON-структурами
  обёрток. Читай, когда нужна команда за пределами ac.py.
- `references/recipes.md` — готовые рецепты: экспликация зон, ведомости
  объёмов, окна/двери, классификации, слои конструкций.
  Включает: надёжный список этажей (`stories`) и их группировку с русскими
  названиями, кросс-валидация зон (`validate-zones`), геометрия из bounding
  boxes (`zone-geometry`), категории зон (`zone-categories`), различие
  `Zone_NetArea`/`Zone_CalculatedArea`, фильтр витражей среди окон, сверка
  округления с ведомостями Archicad, матчинг зон с перекрытиями как
  геометрический fallback, проверка суммы стен зоны, BIM-классификации.
- `references/ru-reports.md` — типовые документы российской практики
  (ГОСТ 21.501, СП 54.13330): экспликация помещений, ведомость отделки,
  спецификация заполнения проёмов, квартирография, ТЭП, экспликация полов.
  Читай при любой задаче «сделай ведомость/спецификацию/экспликацию/ТЭП».
- `references/builtin-properties.md` — 527 встроенных свойств по группам.
  Быстрее искать через `ac.py find-prop <подстрока>`.
- `references/supported_commands.json` — точный список команд, реально
  поддерживаемых сборкой пользователя (результат зондирования).

## Когда свойствам зон нельзя доверять

В реальных проектах зоны часто не пересчитаны, площади — 0 или устаревшие.
Иерархия источников для площади зоны (от надёжного к приближённому):

1. `Zone_NetArea` при `validate-zones` без ошибок → **доверяй**
2. Площадь верхней грани перекрытия под зоной (`General_NetTopSurfaceArea`) →
   геометрический факт, независимый от зон; подходит для суммы по этажу
3. `bbox_area` из `zone-geometry` при `fill_ratio ≥ 0.85` →
   комната прямоугольная, bbox близок к реальности
4. `bbox_area` при `fill_ratio < 0.85` → только порядок величины

Для высоты: `bbox_height` из `zone-geometry` (`Get3DBoundingBoxes`) —
геометрически точен; используй при наличии `height_mismatch` в выводе.

**API не отдаёт полигон зоны** — точная площадь нестандартной формы
из API невозможна. Для полигональной точности нужен IFC-экспорт
(IfcSpace содержит контур; отдельный пайплайн с IfcOpenShell).

## Особенности Archicad 25 (важно, сэкономит время)

- Документация на сайте Graphisoft описывает последнюю версию API. В AC25
  **нет** `GetTypesOfElements`, `GetSelectedElements`, `GetStoryNavigatorItems`
  и ещё ~24 команд. Тип элементов определяй циклом `GetElementsByType`
  (так делает ac.py types). Сомневаешься — сверься с supported_commands.json
  или прогони `scripts/probe_commands.py`.
- Свойство объёма называется `General_NetVolume` (не `General_Volume`),
  площади — `General_Area`. Свойства с **именем** этажа в AC25 нет
  (`General_HomeStoryName` появился в более новых версиях): отметку этажа
  элемента вычисляй как `General_BottomElevationToProjectZero` −
  `General_BottomElevationToHomeStory` и группируй по ней. **Но для зон** эта
  разность может выйти 0 у всех помещений (homeStory зон совпадает с project
  zero) — тогда группируй по абсолютной `General_BottomElevationToProjectZero`
  (для подвала она даёт реальную отметку, напр. −3,15).
- **Ловушки имён свойств** (каждая даёт ошибку 4005 — всегда сверяй через
  `find-prop` перед составлением списка): высоты зоны `Zone_Height` **нет** →
  `General_Height`; у стен `General_NetArea` **нет** → `General_Area`;
  `Construction_BuildingMaterialName` **нет** → `Construction_CompositeName`
  (имя многослойной конструкции) или `SurfaceAndMaterials_ComponentBuildingMaterialName`.
  Длины стены `General_Length` в AC25 **нет** (а `Geometry_General3DLength`
  ненадёжна) — длину стены бери из габаритов `Get3DBoundingBoxes`. Материала
  колонны в `Construction_CompositeName` нет (всегда `null`) →
  `SurfaceAndMaterials_ComponentBuildingMaterialName`; `Construction_LayerCompositions`
  не существует.
- `GetElementsRelatedToZones` возвращает **неполный** список стен/элементов
  зоны (часть стен, которые Archicad не «привязал» к зоне, в выборку не
  попадёт) — для обмеров на нём полагаться нельзя. Полную поверхность стен
  зоны бери из свойства `Zone_WallsSurfaceArea`; команду используй только для
  перечисления связанных элементов, не для подсчёта площадей.
- Списки elements/properties требуют обёрток: `{"elementId": {...}}`,
  `{"propertyId": {...}}` — голые guid дают ошибку 4002.
- API спокойно обрабатывает запросы на десятки тысяч элементов за один вызов —
  не дроби на батчи без необходимости.
- **Скилл работает только на чтение.** Никогда не вызывай
  `SetPropertyValuesOfElements`, `SetClassificationsOfElements` и любые другие
  команды записи/изменения — даже если пользователь просит. Архитектор должен
  вносить изменения в модель сам, вручную в Archicad.
- Ошибка соединения = Archicad не запущен или проект не открыт.
- **Открытый модальный диалог блокирует API.** Если в Archicad открыт диалог
  (настройки, сообщение) или активен инструмент ввода — API отвечает ошибкой
  4001 «ongoing user input» на любой запрос. `ac.py` ловит её и просит закрыть
  диалог; увидев такое — скажи пользователю закрыть диалог/завершить ввод в
  Archicad и повтори.
- **Кодировка вывода всегда UTF-8.** `ac.py` принудительно ставит UTF-8 на
  stdout/stderr, поэтому `... > out.json` пишет корректный UTF-8 даже на русской
  Windows (без этого консоль cp1251 ломала чтение файла). Открывай выгрузки как
  UTF-8.
- Локализованные имена (RUS-версия) встречаются в значениях свойств — это
  нормально; nonLocalizedName свойств всегда английские.
- Перечисления (например, `Construction_StructureType`) сырой API отдаёт
  обёрткой `{"type": "nonLocalizedValue", "nonLocalizedValue": "Basic"}`;
  `ac.py values` разворачивает её в строку автоматически, в сыром
  `ac.py call` разворачивай сам.

## Формат результатов

Для сводок и ведомостей по умолчанию давай компактную таблицу с итогами
(количество, сумма площадей/объёмов) и указывай единицы: длины и объёмы в API —
метры и м³, площади — м². Суммы считай через `math.fsum()` — точное
суммирование float без накопления ошибки (для строительных величин и обычный
`sum()` точен, но fsum снимает вопрос). **Но если итог сверяется с готовой
ведомостью/каталогом Archicad** — округляй каждое значение до 0,01 перед
суммированием (`sum(round(v, 2) ...)`): Archicad суммирует уже округлённые
числа, и сырые float расходятся с ним на ±0,1 м² по зданию (см. recipes.md,
«Сверка итогов с ведомостями Archicad»). Большие выгрузки сохраняй в
JSON/CSV-файл рядом с проектом пользователя и давай ссылку на файл.

**Кириллица в терминале Windows.** Консоль PowerShell/cmd часто рендерит
кириллицу из stdout как `???` (особенно при `python -c "..."`). Это искажение
**только отображения** — не порть из-за него данные. Не полагайся на
кириллический вывод в консоли: перенаправляй результат в файл
(`ac.py … > out.json` — пишет корректный UTF-8) и читай файл через Read-tool,
а не из терминала. Промежуточные Python-скрипты тоже пиши вывод в файл
(`open(path, "w", encoding="utf-8")`), а не в `print`.
